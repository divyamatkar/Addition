Form Validation using JavaScript

Aim:

To create a JavaScript-based form validation program that validates user input fields such as name, email, password, and confirm password before submitting the form.

Objectives:

  • To understand how to validate form fields using JavaScript on the client side.
  • To check mandatory fields like name, email, password, and confirm password.
  • To apply validation rules such as minimum length, pattern matching, and password matching.
  • To display appropriate error and success messages dynamically on the webpage.

Theory:

Form Validation:

Form validation is the process of checking whether the data entered by the user in a form is correct, complete, and acceptable before it is sent to the server. Validation can be:

  • Client-side validation: Done using JavaScript in the browser.
  • Server-side validation: Done on the server after submission.

Client-Side Validation using JavaScript:

JavaScript is commonly used for client-side validation because it:

  • Provides instant feedback to the user.
  • Reduces unnecessary server requests.
  • Improves user experience.

Validation Rules Used in This Program:

Name:

  • Cannot be empty.
  • Must contain only letters and spaces (/^[A-Za-z\s]+$/).
  • Must be at least 3 characters long.

Email:

Checked using a regular expression pattern like ^[^\s@]+@[^\s@]+\.[^\s@]+$ to ensure proper email format.

Password:

  • Must be at least 6 characters long.

Confirm Password:

  • Must match the password field.

DOM Manipulation:

JavaScript uses document.getElementById() to:

  • Read values from input fields.
  • Show error messages inside <div class="error"> elements.
  • Display a success message when all validations pass.

Event Handling:

The submit event of the form is captured using:

document.getElementById('myForm').addEventListener('submit', ...);

event.preventDefault() is used to stop the form from submitting if validation fails.

Conclusion:

By implementing this form validation program using JavaScript, we learned how to verify user inputs on the client side before form submission. This practical helped us understand the use of regular expressions, conditional checks, DOM manipulation, and event handling. It also showed how proper validation improves data quality, user experience, and reduces invalid submissions to the server.
